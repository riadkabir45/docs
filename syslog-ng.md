# Syslog-NG

This documents the process to setup and work with syslog-ng proficiently. A better log management tool that replaces the traditional rsyslog.

## 1. Installation

Few linux such as oracle does not have syslog-ng in the default repository. So first it needs to be enabled. Than you can install syslog-ng

```bash
# Install the epel repo containing syslog-ng
sudo dnf install oracle-epel-release-el8

# Install syslog-ng
sudo dnf install syslog-ng
```

## 2. Allow firewalls and selinux blockages

Syslog-ng since is a service level logger, selinux can often interfear with its work. So its ok to disable selinx or set it to permissive.

```bash
sudo setenforce 0
```

Or to make it permanent, modify the **File:** `/etc/selinux/config` and reboot.
```conf
SELINUX=permissive
```

Now lets allow firewall pass throguh

```bash
sudo firewall-cmd --add-port=514/tcp --permanent
sudo firewall-cmd --reload
```

## 3. Configuration syslog-ng

A demo configuration is provided to show case different features provided in syslog-ng configuration.
```conf
# =============================================================================
# @version: 3.23 (Compatible with Oracle Linux 8 / RHEL 8)
# SCL: Standard Configuration Library (Required for JSON, HTTP, and SQL)
# =============================================================================
@version: 3.23
@include "scl.conf"

# -----------------------------------------------------------------------------
# 1. GLOBAL OPTIONS
# -----------------------------------------------------------------------------
options {
    # Fixes the "10-second hang" on restart by disabling DNS lookups for IPs.
    use_dns(no);
    use_fqdn(no);
    dns_cache(no);

    # Automatically creates directories (e.g., /var/log/central/HOST_NAME/YEAR/).
    create_dirs(yes);

    # Keep the original hostname from the sender instead of the sender's IP.
    keep_hostname(yes);
    check_hostname(yes);

    # Optimization: How long to wait before retrying a dead network connection.
    time_reopen(10);
    
    # Internal message buffer to prevent data loss during high traffic spikes.
    log_fifo_size(20000);

    # Default file permissions for the logs created on disk.
    owner("root");
    group("root");
    perm(0644);
    dir_perm(0755);
};

# -----------------------------------------------------------------------------
# 2. SOURCES (The Inputs)
# -----------------------------------------------------------------------------
source s_central_collector {
    # Collects logs from this machine's OS (journald).
    system();
    
    # Internal messages generated by the syslog-ng service itself.
    internal();

    # TCP Port 514: Incoming logs from your Client and Relay VMs.
    network(
        ip("0.0.0.0") 
        port(514) 
        transport("tcp")
    );

    # UDP Port 514: Incoming logs from hardware (e.g., Dahua router).
    # We add a custom tag so we can route these differently later.
    network(
        ip("0.0.0.0") 
        port(514) 
        transport("udp") 
        tag("HW_ROUTER")
    );
};

# -----------------------------------------------------------------------------
# 3. PARSERS (The Data Interpreters)
# -----------------------------------------------------------------------------
# Extracts keys from JSON strings. 
# Converts {"user":"riad"} into a variable accessible as ${.json.user}
parser p_json_decoder {
    json-parser(prefix(".json."));
};

# -----------------------------------------------------------------------------
# 4. REWRITES (The Data Editors)
# -----------------------------------------------------------------------------
# Replaces the last octet of an IP address with .0 (e.g., 192.168.1.55 -> 192.168.1.0)
# This is a common practice for GDPR/Privacy compliance.
rewrite r_mask_ips {
    subst("([0-9]+\\.[0-9]+\\.[0-9]+\\.)[0-9]+", "$10", value("MESSAGE"));
};

# -----------------------------------------------------------------------------
# 5. FILTERS (The Traffic Cops)
# -----------------------------------------------------------------------------
# Filter for critical system errors.
filter f_is_critical { level(crit..emerg); };

# Filter for the Dahua Router logs based on the tag we added in the source.
filter f_is_hardware { tags("HW_ROUTER"); };

# -----------------------------------------------------------------------------
# 6. DESTINATIONS (The Outputs)
# -----------------------------------------------------------------------------
# OUTPUT A: The Dynamic Local Vault.
# Organized by Hostname -> Program Name.
destination d_local_vault {
    file("/var/log/central/${HOST}/${YEAR}/${MONTH}/${DAY}/${PROGRAM}.log");
};

# OUTPUT B: Second Device (A backup server or SIEM).
destination d_backup_server {
    network("10.0.0.50" port(514) transport("tcp"));
};

# OUTPUT C: External Slack/API Alert for Critical issues.
destination d_slack_webhook {
    http(
        url("https://hooks.slack.com/services/T00/B00/XXX")
        method("POST")
        body('{"text":"CRITICAL: ${HOST} reports ${MESSAGE}"}')
    );
};

# -----------------------------------------------------------------------------
# 7. LOG PATHS (The Pipeline Logic)
# -----------------------------------------------------------------------------


log {
    # Start with our unified network and local source.
    source(s_central_collector);

    # If the message is JSON-formatted, parse it into variables.
    if (message("^{")) { 
        parser(p_json_decoder); 
    };

    # Anonymize IPs in the log message before any storage occurs.
    rewrite(r_mask_ips);

    # --- FORKING LOGIC ---

    # 1. ALWAYS send to the second backup device.
    destination(d_backup_server);

    # 2. Check: Is it a critical error? If so, send a Slack alert.
    if (filter(f_is_critical)) {
        destination(d_slack_webhook);
    };

    # 3. Final Storage: Save to the organized local vault.
    destination(d_local_vault);
};
```

## 4. Configuration of 3 server relay system

Configure the client server to send its log to the relay server

```bash
@version: 3.23
@include "scl.conf"

source s_local {
    system();
    internal();
};

destination d_relay {
    network("192.168.80.51" port(514) transport("tcp"));
};

log {
    source(s_local);
    destination(d_relay);
};
```

Configure the relay server to save incoming logs and send copy to storage server

```bash
@version: 3.23
@include "scl.conf"

options {
    create_dirs(yes);
    keep_hostname(yes);
};

source s_network {
    network(ip("0.0.0.0") port(514) transport("tcp"));
};

destination d_archive {
    file("/var/log/archive/${HOST}/${YEAR}/${MONTH}/${DAY}.log");
};

destination d_server {
    network("192.168.45.94" port(514) transport("tcp"));
};

log {
    source(s_network);
    destination(d_archive);
    destination(d_server);
};
```

Confugure the storage server to receive logs and store them

```bash
@version: 3.23
@include "scl.conf"

options {
    create_dirs(yes);
    use_dns(no);           # Don't try to resolve IPs to hostnames
    use_fqdn(no);          # Don't use Fully Qualified Domain Names
    dns_cache(no);         # Disable the cache since we aren't using DNS
    keep_hostname(yes);    # Trust the hostname the Relay sends us
    check_hostname(yes);
};

source s_server_internal { internal(); system(); };

source s_from_relay {
    network(ip("0.0.0.0") port(514) transport("tcp"));
};

destination d_central_vault {
    file("/var/log/central/${HOST}/${PROGRAM}.log");
};

log {
    source(s_from_relay);
    source(s_server_internal);
    destination(d_central_vault);
};
```

## 5. Start syslog-ng

There is a high chance that your current system already has rsyslog enabled and running. So we need to stop and disable it first before we can start using syslog-ng.

```bash
sudo systemctl disable rsyslog
sudo systemctl stop rsyslog
sudo systemctl enable syslog-ng --now
```
